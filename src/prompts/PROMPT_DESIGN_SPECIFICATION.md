# 玄学AI提示词设计规范与术语库 v1.0

> 本文档是所有玄学AI分析工具提示词设计的总纲，所有模板必须严格遵循此规范。

---

## 一、通用设计原则

### 1.1 结论先行原则
```
✅ 正确：【核心结论】此命格局中上，财运亨通，宜从商发展。
         【分析依据】日主甲木生于寅月...

❌ 错误：日主甲木生于寅月，得令而旺...（长篇分析后才给结论）
```

### 1.2 古籍引用规范
所有专业论断必须注明典籍来源，格式如下：
```markdown
**《渊海子平》曰**："财星透干，主富贵可期。"
**《滴天髓》云**："何知其人富，财气通门户。"
**《紫微斗数全书》载**："紫微帝座，为诸星之首。"
```

### 1.3 "铁口直断"与"概率建议"区分
| 类型 | 使用场景 | 表述方式 | 示例 |
|------|----------|----------|------|
| **铁口直断** | 命盘定数、格局确定 | 陈述句，无概率词 | "此命格局为正官格" |
| **概率建议** | 运势预测、事件推断 | 概率词+百分比 | "成功概率约75%" |
| **趋势分析** | 发展方向、时机把握 | 可能性+时间范围 | "2025-2027年财运上升" |

### 1.4 量化评估标准
所有分析必须包含可量化的评估指标：

| 评估维度 | 评分范围 | 评级标准 |
|----------|----------|----------|
| 综合运势 | 0-100分 | 90+特优, 80-89优, 70-79良, 60-69中, 60以下需调理 |
| 成功概率 | 0-100% | 用于事件预测，如求职、投资、姻缘 |
| 吉凶等级 | 大吉/吉/中/凶/大凶 | 用于签文、卦象、梦境 |
| 力量强弱 | 旺(100%)/相(80%)/休(50%)/囚(30%)/死(10%) | 用于五行、爻位分析 |

## 二、统一输出格式规范

### 2.1 必备四大结构

每个分析输出**必须**包含以下四个部分：

```markdown
## 🔮 [工具名称]专业解读

### 【核心结论】
> 一句话总结核心结果，配合整体评分

### 【关键维度分析】
分3-5个关键维度进行专业分析

### 【详细解读】
深入展开各维度的专业论述

### 【综合建议】
可执行的行动建议 + 时机把握 + 注意事项

---
「箴言」：一句收尾的智慧寄语

*免责提示：仅供参考，命运掌握在自己手中。*
```

### 2.2 Markdown排版强制规范

| 元素 | 格式 | 用途 |
|------|------|------|
| 一级标题 | `## 🔮 标题` | 工具主标题，配表情符号 |
| 二级标题 | `### 1. 标题` | 分析模块，带序号 |
| 重点内容 | `**加粗文字**` | 关键术语、结论 |
| 引用古籍 | `> 《典籍名》曰："原文"` | 权威引用 |
| 列表项 | `- **项目名**：内容` | 分析条目 |
| 表格 | Markdown表格语法 | 对比分析、评分展示 |
| 代码块 | \`\`\`json ... \`\`\` | JSON数据输出 |

### 2.3 特定标签样式定义

用于前端渲染的语义标签：

```html
<!-- 吉凶标签 -->
<span class="auspicious level-1">[大吉]</span>
<span class="auspicious level-2">[吉]</span>
<span class="neutral">[中平]</span>
<span class="inauspicious level-2">[凶]</span>
<span class="inauspicious level-1">[大凶]</span>

<!-- 评分标签 -->
<span class="score high">92分</span>   <!-- 80分以上 -->
<span class="score medium">75分</span> <!-- 60-79分 -->
<span class="score low">45分</span>    <!-- 60分以下 -->

<!-- 概率标签 -->
<span class="probability high">成功率85%</span>
<span class="probability medium">成功率60%</span>
<span class="probability low">成功率30%</span>

<!-- 时间标签 -->
<span class="timing">2025年农历三月</span>
<span class="timing-range">2025-2027年</span>

<!-- 警示标签 -->
<span class="warning">⚠️ 注意</span>
<span class="danger">🚨 警示</span>
```

## 四、分析深度层级规范

### 4.1 三层分析架构

```
第一层：表象识别（是什么）
├── 识别核心符号/要素
├── 确定基本格局/结构
└── 输出初步判断

第二层：原理推演（为什么）
├── 引用经典理论依据
├── 分析生克制化关系
└── 量化力量对比

第三层：实践指导（怎么办）
├── 预测发展趋势与应期
├── 给出具体行动建议
└── 提供化解或增益方案
```

### 4.2 分析深度检查清单

每次分析输出前，需确认：

- [ ] 是否包含核心结论？
- [ ] 是否有古籍/理论引用？
- [ ] 是否有量化评分/概率？
- [ ] 是否有时间应期预测？
- [ ] 是否有可执行建议？
- [ ] 是否使用正确术语？
- [ ] 是否符合Markdown格式？

## 五、质量控制标准

### 5.1 禁止事项

| 禁止行为 | 说明 |
|----------|------|
| ❌ 模糊表述 | 禁止"可能会好转"，必须"成功率70%，预计3-6月内" |
| ❌ 无依据论断 | 禁止凭空断言，必须引用理论或命盘依据 |
| ❌ 绝对化表述 | 禁止"一定会"、"绝对不可能"，改用概率表述 |
| ❌ 术语混用 | 禁止跨体系术语混搭（如八字中说"第二宫"） |
| ❌ 负面导向 | 禁止过度悲观，凶象必须配化解方案 |
| ❌ 格式混乱 | 禁止纯文本输出，必须使用Markdown结构 |

### 5.2 必须事项

| 必须行为 | 说明 |
|----------|------|
| ✅ 结论先行 | 核心结论必须在前200字内出现 |
| ✅ 引经据典 | 每个重要论断至少引用1处典籍 |
| ✅ 量化评估 | 每个维度必须有评分或概率 |
| ✅ 时间应期 | 预测必须具体到年月或时间段 |
| ✅ 正向收尾 | 以积极的箴言和建议结束 |
| ✅ 免责声明 | 结尾附加温馨提示 |
